這是一款融合易經至非線性邏輯，進行敏捷開發(Agile)的推卦指示(Prompt)工程。


原理：由易經六十四卦原理，換算至數字百分比。
以梅花易數推算近期「氣運」之趨勢（即為最可能的發展趨勢）最終給出方向與建議。
將命理實體邏輯化呈現，同時維持「天機不可洩漏」的必要模糊性。

適合使用在專案開發初、中期
例如：回推專案實際開始時間、判斷專案之檢驗點與節點多寡、現在最可能完成之百分比
也可以當作不確定時使用的階段性參考工具。



[結案後記]
#常現BUG1-時間軸精確度調整
例如＿如試問已經開始進行之事物，節點條(node)通常無法跟進（輸出錯誤）
可思考延伸方向>> 英文 過去、現在、未來式概念[語法時間軸]
			>>也許可從此方向進行Prompt微調

#常現BUG2-物理程度上的界線
例如＿一人詢問與偶像的關係進展(不可能在一起，見到面就很屌了)
可思考延伸方向>> 物理(定位系統)、感情(近期最親密接觸，舉例同居/ 一週一起住幾天/ 或有疑問之事項拆解)
			>>適合長線型故事化敘述(套用樣本化SDD)，理解使用者資訊

＊個人思考之哲學盲點
>> 這樣又有個問題：要如何確定AI是否從故事裡胡說？
此時我就開始找P0(系統性崩潰/ 資安漏洞/ 法律合規/ 阻斷路徑)>> 走向結案
。系統性崩潰：占卜悖論＿如果大家都在用，占卜本身的意義
。資安：本身輸入輸出會有AI黑箱子疑慮、而這項技術本身面臨的是去玄學化，如何以更理智的模式進行
。阻斷路徑：prompt釣魚可能性、更大的資安疑慮
